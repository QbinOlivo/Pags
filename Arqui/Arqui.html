Unidad 1: Arquitectura de Cómputo

1.1 Modelos de arquitecturas de cómputo
  • Arquitectura Von Neumann:
    • Un solo bus compartido para datos e instrucciones.
    • Ventajas: Simplicidad y bajo costo.
    • Desventaja: Cuello de botella en el bus (no puede acceder a datos e instrucciones al mismo tiempo).
    • Ejemplo: Computadoras personales, laptops, servidores tradicionales.
  • Arquitectura Harvard:
    • Buses separados para datos e instrucciones.
    • Ventajas: Mayor velocidad y eficiencia, permite acceso simultáneo a datos e instrucciones.
    • Ejemplo: Microcontroladores, DSPs, sistemas embebidos.
  • Arquitectura Pipeline:
    • Divide la ejecución de instrucciones en etapas secuenciales (fetch, decode, execute, write back).
    • Permite procesar varias instrucciones a la vez (paralelismo a nivel de instrucción).
    • Ventaja: Aumenta el rendimiento sin aumentar la frecuencia de reloj.
    • Desventaja: Riesgo de conflictos y burbujas en el pipeline.
  • Arquitecturas multiprocesador:
    • SISD (Single Instruction, Single Data): Computadoras tradicionales.
    • SIMD (Single Instruction, Multiple Data): GPUs, procesamiento vectorial.
    • MISD (Multiple Instruction, Single Data): Poco común, sistemas de tolerancia a fallos.
    • MIMD (Multiple Instruction, Multiple Data): Multiprocesadores, clusters, supercomputadoras.
1.2 Análisis de componentes
  • CPU: Unidad de control (coordina operaciones), ALU (realiza cálculos aritméticos y lógicos), registros (almacenamiento temporal de datos).
  • ALU (Unidad Aritmético Lógica): Realiza operaciones aritméticas (suma, resta, multiplicación, división) y lógicas (AND, OR, NOT, XOR).
  • Registros: Memoria interna rápida para operaciones inmediatas. Ejemplo: AX, BX en x86; R0-R15 en ARM.
  • Buses:
    • Bus de datos: Transfiere información.
    • Bus de direcciones: Indica destino/origen.
    • Bus de control: Gestiona operaciones.
  • Memoria:
    • Jerarquía: registros → caché (L1, L2, L3) → RAM → almacenamiento secundario (discos, SSDs).
    • Tipos: SRAM (rápida, cara, caché), DRAM (lenta, barata, RAM principal), ROM, memorias flash.
  • Entrada/Salida:
    • E/S programada: CPU controla directamente la transferencia de datos.
    • E/S por interrupciones: Dispositivos notifican al CPU cuando están listos.
    • DMA: Transferencia directa a memoria sin intervención del CPU.
  • Interrupciones: Permiten a dispositivos notificar eventos al CPU (hardware/software).

Unidad 2: Unidad Central de Procesamiento

2.1 Organización del procesador
  • Núcleos de ejecución (multicore): Cada núcleo puede ejecutar instrucciones de manera independiente.
  • Unidad de Manejo de Memoria (MMU): Traduce direcciones lógicas a físicas.
  • Memoria caché (L1, L2, L3): Almacena datos e instrucciones de uso frecuente.
  • Controlador de memoria: Gestiona el acceso a la RAM.
  • Unidades de ejecución especializadas: FPU (punto flotante), SIMD, GPU integrada.
2.2 Estructura de registros
  • Registros de propósito general: Almacenamiento temporal de datos (EAX, EBX en x86; R0-R15 en ARM).
  • Registros de punteros e índices: Gestión de pila (ESP, EBP) y operaciones con cadenas (ESI, EDI).
  • Registros de control y estado:
    • Contador de programa (PC): Dirección de la siguiente instrucción.
    • Registro de instrucciones (IR): Instrucción actual.
    • Registro de banderas (FLAGS): Estado del procesador (zero, carry, sign, overflow).
2.3 El ciclo de la instrucción
  1. Fetch: Obtención de la instrucción desde memoria.
  2. Decode: Decodificación de la instrucción.
  3. Execute: Ejecución de la operación.
  4. Write-back: Almacenamiento del resultado.
2.3.2 Segmentación de instrucciones
Ejemplo de pipeline de 5 etapas:
IF (Instruction Fetch) → ID (Instruction Decode) → EX (Execute) → MEM (Memory Access) → WB (Write Back)
2.3.3 Conjunto de instrucciones (ISA)
  • Aritméticas (ADD, SUB, MUL, DIV)
  • Lógicas (AND, OR, XOR, NOT)
  • Transferencia de datos (MOV, LOAD, STORE)
  • Control de flujo (JMP, CALL, RET)
  • Instrucciones especializadas (SIMD, cifrado, multimedia)
2.3.4 Modos de direccionamiento
  • Inmediato: Operando incluido en la instrucción.
  • Directo: Dirección de memoria especificada.
  • Indirecto: Registro contiene la dirección.
  • Registro: Operando en registro.
  • Indexado: Dirección = registro + desplazamiento.
  • Base + desplazamiento: Similar a indexado.
  • Relativo al PC: Para saltos condicionales.

Unidad 3: Selección de Componentes

3.1 Chipset y componentes clave
  • Northbridge: Controla memoria y gráficos (RAM, GPU).
  • Southbridge: Gestiona dispositivos de E/S (USB, SATA, audio, red).
  • VRM (Módulo Regulador de Voltaje): Proporciona alimentación estable al CPU.
  • Conectores: ATX, EPS, PCIe, SATA, M.2.
  • Slots de expansión: PCI Express x1, x4, x16.
  • Criterios de selección:
    • Compatibilidad de socket (Intel, AMD, etc.).
    • Tipo y velocidad de RAM soportada.
    • Conectividad (USB, SATA, M.2, WiFi, Bluetooth).
    • Soporte para overclocking y refrigeración.
3.2 Aplicaciones y ambientes de servicio
  • Entornos empresariales:
    • Fiabilidad y redundancia (RAID, fuentes dobles).
    • Sistemas RAID para respaldo de datos.
    • Equipos All-in-One para ahorrar espacio.
  • Entornos industriales:
    • Resistencia a condiciones extremas (temperatura, polvo, vibración).
    • Conectores industriales (RS-485, CAN bus).
    • Soporte a largo plazo y mantenimiento sencillo.
  • Comercio electrónico:
    • Alta disponibilidad (24/7), escalabilidad, redundancia.
    • Almacenamiento rápido (SSD NVMe, RAID).
  • Gaming y multimedia:
    • GPU dedicada de alto rendimiento.
    • Refrigeración avanzada (líquida, aire).
    • Soporte para overclocking.
    • Conectividad de alta velocidad (Ethernet, WiFi 6).

Unidad 4: Procesamiento Paralelo

4.1 Aspectos básicos
  • División de problemas en partes ejecutables simultáneamente (paralelismo).
  • Ley de Amdahl: Límite teórico de mejora mediante paralelismo.
  • Desafíos:
    • Sincronización de procesos.
    • Coherencia de caché.
    • Contención de recursos.
    • Balanceo de carga.
4.2 Tipos de computación paralela
  • Clasificación Flynn:
    • SISD (Single Instruction, Single Data): Computadoras tradicionales.
    • SIMD (Single Instruction, Multiple Data): GPUs, procesamiento vectorial.
    • MISD (Multiple Instruction, Single Data): Poco común.
    • MIMD (Multiple Instruction, Multiple Data): Multiprocesadores, clusters.
  • Granularidad:
    • Paralelismo a nivel de bit, instrucción, datos, tarea.
4.3 Sistemas de memoria compartida
  • UMA (Uniform Memory Access): Tiempo de acceso uniforme para todos los procesadores.
  • NUMA (Non-Uniform Memory Access): Tiempo de acceso variable según la ubicación de la memoria.
  • Memoria distribuida: Cada procesador tiene memoria local.
  • Modelos híbridos: Combinación de memoria compartida y distribuida.
4.4 Redes de interconexión
  • Topologías estáticas:
    • Malla 2D/3D
    • Tórus
    • Hipercubo
    • Árbol binario
  • Topologías dinámicas:
    • Bus compartido
    • Crossbar
    • Red multietapa
4.5 Casos de estudio y tecnologías
  • Supercomputadoras:
    • Frontier (EE.UU.): Más de 1 exaflop, arquitectura híbrida CPU-GPU.
    • Fugaku (Japón): Basada en ARM, alto rendimiento y eficiencia energética.
    • LUMI (Europa): Enfocada en IA y simulaciones científicas.
  • Tecnologías paralelas:
    • CUDA (NVIDIA): Programación paralela en GPUs.
    • OpenCL (estándar abierto): Computación paralela en CPUs, GPUs y FPGAs.
    • ROCm (AMD): Plataforma abierta para GPUs AMD.
    • oneAPI (Intel): Unifica programación para CPUs, GPUs y FPGAs.
  • Frameworks:
    • MPI (Message Passing Interface): Comunicación entre procesos en memoria distribuida.
    • OpenMP (para memoria compartida): Paralelismo en CPUs multinúcleo.
  • Tendencias futuras:
    • Computación heterogénea (CPU+GPU+FPGA).
    • Aceleradores especializados (IA, ML, criptografía).
    • Interconexiones ópticas para mayor velocidad.
    • Arquitecturas neuromórficas (inspiradas en el cerebro).
